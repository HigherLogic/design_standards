Error Handling

Error handling provides feedback to the user when an input error occurs. There are mulitple parts to handling an error: the required indicator on the form field, visual feedback on the input element that failed validation and help text or messaging for the user explaining how to correct the error.

NOTE: Required fields must be indicated as described below and have concise and germain help block text.

Steps

  • Add required class to form-group div, this adds the asterisk.
    Code
  • Add a span classed with help-block after the required form input and within the form-group.
    Code
  • When validation fails and an error occurs, dynamically add the has-error class to the form-control div and Bootstrap automatically shows the help-block span with the error text. The error text should clearly state what is needed to meet the requirement. It should be as concise and precise as needed to ensure successful input by the user.

    All together it should look like this.
    Code
  • When the user is successful and an error state no longer exists, remove the has-error class.
  • Read Bootstrap's documentation on validation states.

Other Validation State Examples
